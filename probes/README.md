## Required Fields Per Probe

1. **Query** — the natural language question used to trigger the failure
2. **Failure category** — one of the 4 DAB categories
3. **Expected failure** — what the agent does wrong before the fix
4. **Observed failure** — what actually happened during live testing
5. **Fix applied** — specific code/doc change with post-fix score

## How Probes Were Created

Probes were generated by the Intelligence Officers from:
- Known DAB failure modes (documented in `kb/architecture/dab_failure_modes.md`)
- Live failures observed during benchmark runs (cross-referenced with `kb/corrections/corrections_log.md`)
- Anti-patterns documented in `kb/domain/yelp_antipatterns.md`

Post-fix scores are verified against actual benchmark runs where available (probes 016–020 verified against Yelp benchmark run 20260415_183320).
